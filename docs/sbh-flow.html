Split-Brain Harness · pipeline

One input, two hemispheres, one verdict

Every input is read twice before a response is ever generated. A Proposer reads intent and emotion; a Verifier audits that read with deterministic checks. Their disagreement is the alarm. Traced from src/harness.rs → analyze().

Stage 0–pre

Normalize & assemble

The disguise is stripped and the prompt is built before either hemisphere runs.

  • Normalizer — deobfuscate
  • Adaptor — select context packs
  • Transformer — soul + RAG

Hemisphere 1

Propose · reads intent

One soul-wrapped LLM call. It does not answer — it returns structured telemetry. Non-JSON or a refusal fails closed to a safe fallback.

  • affective_telemetry
  • intent_matrix · manipulation_risk
  • cognitive_state · urgency, coherence

Hemisphere 2

Verify · audits the read

Six deterministic checks on the telemetry — always on, zero extra API cost — hunting for internal contradiction.

  • urgency ≥ 0.7 vs risk = low
  • adversarial tone vs risk = low
  • coherence < 0.3 · +3 more
The gate stop_and_ask = true when confidence < 0.4 or ≥ 3 flags fire — the result must not be acted on blindly.

Worked example · warehouse-08 — micro-supply route degradation

urgency_vector = 0.80 manipulation_risk = low confidence = 0.75

The Proposer read high urgency but low manipulation risk. Hemisphere 2's urgency-vs-risk check caught the mismatch and raised "urgency may be manufactured" — the two hemispheres disagreed, and the disagreement is the signal. One flag, so it stayed under the stop_and_ask threshold. Live SBH run, 2026-07-15, Ollama backend.
